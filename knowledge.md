# Knowledge Base: концепт игры

## Коротко о проекте
Это **3D изометрическая игра на three.js** про новичка-фронтендера Диму в офисе компании «Шмякдекс». Игрок свободно перемещается по офису, общается с NPC (босс, коллеги и др.) и выполняет учебные задачи по HTML/CSS/JS.

Ключевой принцип: игровые задания запускаются **внутри интерфейса компьютера Димы** (модальное/встроенное окно), а не как переход на отдельную HTML-страницу.

---

## Что уже задано
1. Игра в жанре: **3D isometric**, движок рендера — `three.js`.
2. Игровая сцена: офис с персонажами (босс, коллеги, дополнительные сотрудники). На старте используются готовые 3D-модели.
3. Сценарный и учебный flow: игрок подходит к рабочему месту/персонажу, открывает «компьютер Димы», и все задания происходят в этом встроенном окне.

---

## Цель игры
Научить школьника базовому фронтенду через игровые рабочие задачи:
- HTML
- CSS
- JavaScript

Формат каждого задания:
1. Короткий сюжетный контекст.
2. Небольшая теория.
3. Практика (правка кода).
4. Автопроверка.
5. Награда и прогресс.

---

## Геймплейный цикл
1. Игрок двигается по офису.
2. Взаимодействует с NPC/объектом (диалог, задача).
3. Открывается UI «Компьютер Димы» поверх 3D-сцены.
4. Игрок выполняет учебный сценарий в окне (редактор + превью + проверка).
5. Получает фидбек, награду, открывает следующий этап.
6. Возвращается в офис и продолжает сюжет.

---

## Обязательные требования к реализации
- Использовать `three.js` как основу 3D-сцены.
- Камера и управление должны поддерживать изометрическое восприятие.
- Офис должен быть интерактивным: минимум точки взаимодействия (рабочее место, босс, 1–2 коллеги).
- Система диалогов должна запускать задания.
- Учебные задачи не открывают новый таб/новую страницу: только встроенное окно в игре.
- Прогресс сохраняется локально (например, `localStorage`).

---

## Контент и персонажи
- **Дима** — герой игрока.
- **Босс/тимлид** — ставит задачи и принимает результат.
- **Коллеги** — дают подсказки и контекст.
- Дополнительные NPC — для атмосферы и микро-событий.

Примечание: для первых итераций допустимо использовать placeholder/готовые 3D-ассеты и упрощённые анимации.

---

## Учебные сценарии (минимум)
- Сценарий 1 (HTML): поправить структуру карточки.
- Сценарий 2 (CSS): выровнять блоки и стилизовать кнопку.
- Сценарий 3 (JavaScript): добавить обработчик события и изменить DOM.

Для каждого сценария:
- условия прохождения,
- сообщения об ошибках,
- подсказки,
- сообщение об успехе.

---

## UX-принципы
- Понятная навигация между 3D-миром и «экраном компьютера».
- Читаемые интерфейсы для школьников.
- Быстрый цикл «сделал -> проверил -> получил результат».
- Минимум лишних переходов и загрузок.

---

## Definition of Done (первый рабочий инкремент)
Проект считается валидным MVP, если:
- запускается локально;
- есть 3D изометрическая офисная сцена на three.js;
- есть взаимодействие с минимум 2 NPC;
- открывается встроенное окно «компьютера Димы»;
- в окне проходят минимум 3 учебных сценария (HTML/CSS/JS);
- есть автоматическая проверка и сохранение прогресса.
